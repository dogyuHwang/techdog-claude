# /tdc-onboard — Project Onboarding

기존 프로젝트에 tdc를 도입할 때, 프로젝트의 기술 스택·구조·컨벤션을 자동 분석하여 `.tdc/project-memory.md`에 저장합니다.

**이후 모든 tdc 세션에서 이 지식을 활용하여 코드 품질이 향상됩니다.**

## 실행 흐름

### 1단계: 기존 상태 확인

1. `.tdc/project-memory.md`가 이미 있는지 확인
2. 있으면 → 사용자에게 "이미 온보딩되어 있습니다. 재분석하시겠습니까?" 확인
3. 없으면 → 바로 분석 시작

### 2단계: 프로젝트 분석

Explore Agent를 사용하여 다음을 자동 분석한다:

**기술 스택 감지** (파일 존재 여부로 판단):
- `package.json` → Node.js/TypeScript (dependencies에서 프레임워크 감지)
- `requirements.txt` / `pyproject.toml` / `setup.py` → Python
- `go.mod` → Go
- `Cargo.toml` → Rust
- `build.gradle` / `pom.xml` → Java/Kotlin
- `pubspec.yaml` → Flutter/Dart
- `next.config.*` → Next.js
- `manage.py` + `settings.py` → Django
- `angular.json` → Angular
- `vite.config.*` → Vite

**코딩 컨벤션 감지**:
- `.eslintrc*` / `.prettierrc*` → 린터/포매터 규칙 요약
- `.editorconfig` → 탭/스페이스, 줄바꿈 규칙
- `tsconfig.json` → TypeScript 설정 (strict 등)
- 기존 코드에서 샘플링 → 네이밍 패턴 (camelCase/snake_case), import 스타일 (ESM/CJS)

**프로젝트 구조 감지**:
- 최상위 디렉토리 레이아웃
- 엔트리 포인트 위치 (main.*, index.*, app.*)
- 테스트 디렉토리 + 프레임워크 (jest, pytest, go test 등)
- CI/CD (`.github/workflows/`, `Jenkinsfile`, `.gitlab-ci.yml`)

**빌드/실행 명령 감지**:
- `package.json` scripts 섹션
- `Makefile` 타겟
- `docker-compose.yml` 서비스
- `.env.example` 환경변수

### 3단계: project-memory.md 생성

분석 결과를 `.tdc/project-memory.md`에 저장:

```
# Project Memory — Auto-generated by /tdc onboard
# Last updated: {날짜}
# Re-run: /tdc onboard

## Tech Stack
- Language: {언어} ({버전 if available})
- Framework: {프레임워크}
- Styling: {스타일링 도구 if applicable}
- DB: {데이터베이스 + ORM if applicable}
- Test: {테스트 프레임워크}

## Conventions
- Import style: {ESM/CJS}
- Naming: {camelCase/snake_case/etc}
- Formatter: {포매터 + 핵심 설정}
- Linter: {린터 + 핵심 규칙}

## Project Structure
- {주요 디렉토리 설명}

## Build & Test
- Build: `{빌드 명령}`
- Test: `{테스트 명령}`
- Lint: `{린트 명령}`
- Dev: `{개발 서버 명령}`

## Business Rules
(수동으로 추가 — /tdc onboard는 코드에서 추론할 수 없는 비즈니스 규칙을 여기에 추가하도록 안내합니다)
```

## 토큰 최적화
- Explore Agent는 파일 존재 여부 위주로 빠르게 스캔 (내용은 최소한만 읽기)
- 설정 파일은 핵심 필드만 추출 (전체 읽기 X)
- 결과는 간결한 마크다운으로 출력
